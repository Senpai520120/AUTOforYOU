# Инструкция по запуску AUTOforYOU в продакшене

Написано простым языком — без программного жаргона.

---

## Часть 1. Резервная копия (ВСЕГДА перед изменениями)

**Шаг 1.** Откройте терминал в папке проекта и выполните:
```
python manage.py dumpdata --indent 2 > backups/data_СЕГОДНЯШНЯЯ_ДАТА.json
```
Например: `backups/data_20260622.json`

**Шаг 2.** Скопируйте файл `db.sqlite3` в папку `backups/`:
```
copy db.sqlite3 backups\db_СЕГОДНЯШНЯЯ_ДАТА.sqlite3
```

> ⚠ Папка `backups/` никогда не попадёт в git — это правильно, там личные данные.

---

## Часть 2. Файл настроек (.env)

Все секреты и настройки хранятся в файле `.env` в корне проекта.
Этот файл **не виден в git** — нужно создать его вручную.

**Шаг 1.** Скопируйте шаблон:
```
copy .env.example .env
```

**Шаг 2.** Откройте `.env` в блокноте и заполните нужные строки (смотрите части ниже).

---

## Часть 3. Минимальный .env для локальной разработки

Для запуска на компьютере достаточно двух строк в `.env`:

```
SECRET_KEY=любая-случайная-строка-длиной-50-символов
DEBUG=true
```

Чтобы сгенерировать хороший SECRET_KEY:
```
python -c "from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())"
```

После этого `python manage.py runserver` работает с SQLite — база данных в файле `db.sqlite3`.

---

## Часть 4. Подключить PostgreSQL (для сервера/продакшена)

**Когда нужно:** при деплое на сервер (Heroku, Railway, VPS).
**Когда НЕ нужно:** при локальной разработке — SQLite работает сам.

**Шаг 1.** Создайте базу данных PostgreSQL у вашего провайдера.
Вы получите строку подключения вида:
```
postgres://имя_пользователя:пароль@хост:5432/имя_базы
```

**Шаг 2.** Добавьте в `.env`:
```
DATABASE_URL=postgres://имя_пользователя:пароль@хост:5432/имя_базы
```

**Шаг 3.** Создайте таблицы в новой базе:
```
python manage.py migrate
```

**Шаг 4.** Загрузите данные из резервной копии (сделайте бэкап по Части 1 перед этим!):
```
python manage.py loaddata backups/data_ДАТА.json
```

**Откат к SQLite:** просто уберите строку `DATABASE_URL=` из `.env` (или закомментируйте знаком `#`).

## Часть 5. Подключить хранилище фото S3 (для продакшена)

**Когда нужно:** когда сайт работает на сервере и нужно хранить фото надёжно.
**Когда НЕ нужно:** при локальной разработке — фото хранятся в папке `media/`.

### Вариант А: AWS S3

**Шаг 1.** Создайте бакет (bucket) в AWS S3.

**Шаг 2.** Добавьте в `.env`:
```
S3_BUCKET_NAME=название-вашего-бакета
S3_REGION=eu-central-1
AWS_ACCESS_KEY_ID=ваш_ключ_доступа
AWS_SECRET_ACCESS_KEY=ваш_секретный_ключ
```

### Вариант Б: Cloudflare R2 (дешевле, совместим с S3)

```
S3_BUCKET_NAME=название-бакета
S3_REGION=auto
AWS_ACCESS_KEY_ID=ваш_ключ_r2
AWS_SECRET_ACCESS_KEY=ваш_секрет_r2
S3_ENDPOINT_URL=https://ACCOUNT_ID.r2.cloudflarestorage.com
```

**Шаг 3.** Перенесите уже загруженные фото лотов в S3:
```
python manage.py upload_lot_photos
```
Для тестового запуска (без реального скачивания):
```
python manage.py upload_lot_photos --dry-run
```

**Как проверить, что фото работают:**
1. Откройте любое объявление в браузере: `http://ваш-домен/listings/`
2. Нажмите на любое объявление с фото
3. Фото должно загрузиться — в адресной строке браузера будет ссылка на S3 (содержит `.amazonaws.com` или `.r2.cloudflarestorage.com`)

**Откат к локальному хранилищу:** уберите S3-переменные из `.env`.

## Часть 6. Деплой на сервер (чеклист)

1. Задайте `DEBUG=false` в `.env`
2. Задайте `SECRET_KEY=` (обязательно новый, не из примера!)
3. Задайте `ALLOWED_HOSTS=ваш-домен.com`
4. Задайте `DATABASE_URL=` (PostgreSQL)
5. Выполните `python manage.py migrate`
6. Выполните `python manage.py collectstatic --noinput`
7. (если нужно S3) Задайте S3-переменные и запустите `upload_lot_photos`
8. Откройте `https://ваш-домен.com/api/docs/` — Swagger должен работать

---

## Часть 7. Откат (если что-то пошло не так)

### Откат базы данных
1. Остановите сервер
2. Скопируйте резервный файл обратно: `copy backups\db_ДАТА.sqlite3 db.sqlite3`
3. Или для PostgreSQL: `python manage.py loaddata backups/data_ДАТА.json`

### Откат на SQLite
1. Уберите из `.env` строку `DATABASE_URL=`
2. Перезапустите сервер — автоматически переключится на `db.sqlite3`

### Откат на локальные фото
1. Уберите из `.env` строки S3_* и AWS_*
2. Перезапустите сервер — фото будут читаться из папки `media/`

## Часть 8. Postgres — проверить вручную

Если у вас есть Postgres и вы хотите проверить:

```
# Задайте DATABASE_URL в .env, затем:
python manage.py migrate
python manage.py test
```

Все 119 тестов должны быть зелёными.
